# نقشه راه: SNI Spoofer «بن‌بست‌ناپذیر»

سند استراتژی توسعه — تحلیل ۲۰۲۶-۰۵-۲۹

هدف: تبدیل ابزار تک‌ترفندی فعلی (فقط `wrong_seq`) به یک پلتفرم چند‌لایه‌ی تطبیق‌پذیر
که هیچ DPIای نتواند به‌سادگی آن را دیکته یا مسدود کند — یعنی برای آن **بن‌بست معنا نداشته باشد**.

---

## ۱. وضعیت فعلی (نقطه‌ی شروع)

| مؤلفه | وضعیت | محدودیت کلیدی |
|---|---|---|
| `ProxyServer` (forwarder localhost + WinDivert) | ✅ کار می‌کند | فقط Windows، فقط `wrong_seq` |
| `TransparentSpoofServer` | ✅ کار می‌کند | فقط Windows |
| `xray/vwarp/warp managers` | ✅ embedded | باینری‌های Windows |
| GUI | ✅ کامل | tkinter دسکتاپ |
| تکنیک bypass | **یک‌تا**: `wrong_seq` | تک‌نقطه‌ی شکست |

**ضعف بنیادی:** کل بقای ابزار به **یک ترفند** بند است. اگر DPI به wrong_seq حساس شود
(مثلاً با تشخیص segment خارج از پنجره یا الزام پاسخ سرور)، کل ابزار از کار می‌افتد →
این همان «بن‌بست» است.

---

## ۲. فلسفه‌ی «بن‌بست‌ناپذیری»

DPI یک بازی گربه‌وموش است. ابزاری بن‌بست‌ناپذیر است که:

1. **چند‌لایه باشد** — اگر یک تکنیک مسدود شد، ۵ تکنیک دیگر آماده باشد.
2. **خودتطبیق باشد** — به‌صورت خودکار تکنیک مؤثر را کشف و انتخاب کند (auto-probe).
3. **متغیر/تصادفی باشد** — هیچ امضای ثابتی ندهد (هر اتصال کمی متفاوت).
4. **چندپلتفرمی باشد** — Windows + Linux + Android، تا مسدودکردن یک پلتفرم کافی نباشد.
5. **قابل‌به‌روزرسانی از راه دور باشد** — استراتژی‌های جدید بدون آپدیت اپ پخش شوند.

---

## ۳. لایه‌های تکنیک (Arsenal) — از موجود تا پیشرفته

### لایه ۰ — موجود
- `wrong_seq` (تزریق ClientHello جعلی با seq خارج از پنجره). ✅

### لایه ۱ — تکنیک‌های هم‌خانواده‌ی injection (کم‌هزینه، بازده بالا)
1. **`wrong_checksum`** — تزریق fake با checksum خراب. سرور drop می‌کند (checksum بد)،
   اما بعضی DPIها checksum را verify نمی‌کنند و آن را parse می‌کنند. (در کد کامنت شده، فقط فعال‌سازی لازم است.)
2. **`wrong_seq` + `wrong_ack`** — ترکیب چند فیلد نامعتبر برای فریب DPIهای سخت‌گیرتر.
3. **`fake_ttl` / `fake_md5sig`** — fake را با TTL پایین بفرست که به DPI برسد ولی به سرور نه
   (DPI نزدیک‌تر است). یا با TCP MD5 option نامعتبر که فقط سرور رد می‌کند.
4. **multi-fake** — چند ClientHello جعلی با SNIهای مختلف whitelisted، برای گیج‌کردن
   DPIهای حالت‌دار.

### لایه ۲ — تکنیک‌های fragmentation (بدون نیاز به raw socket → قابل‌حمل به اندروید non-root)
5. **TCP segmentation (split ClientHello)** — ClientHello واقعی را در ۲+ سگمنت بفرست،
   نقطه‌ی برش وسط SNI. DPIهای تک‌سگمنتی SNI را نمی‌بینند.
6. **TLS record fragmentation** — خودِ TLS record را به چند record بشکن (مطابق مقاله‌ی
   UPB SysSec علیه GFW). حتی DPIهای بازچیننده‌ی TCP را هم دور می‌زند.
7. **out-of-order segmentation** — سگمنت دوم را قبل از اول بفرست (با کنترل seq روی
   پلتفرم‌های مجاز) → DPIهای بدون بازچینش گیج می‌شوند.
8. **disorder + fake** (سبک zapret `--dpi-desync=fakeddisorder`) — ترکیب fake injection
   با ارسال نامرتب.

### لایه ۳ — دستکاری SNI / TLS
9. **SNI padding/case manipulation** — حروف بزرگ/کوچک، فضای اضافه (بعضی DPIها
   case-sensitive parse می‌کنند ولی سرور نه).
10. **ECH (Encrypted Client Hello)** — اگر سرور پشتیبانی کند، SNI کاملاً رمز می‌شود →
    DPI هیچ‌چیزی برای دیدن ندارد. این **راه‌حل آینده‌ی واقعی** است.
11. **domain fronting (در صورت امکان روی CDN)** — SNI یک دامنه، Host دامنه‌ی دیگر.

### لایه ۴ — لایه‌ی انتقال (وقتی TCP کلاً بسته شد)
12. **QUIC/HTTP3 bypass** — وقتی TCP/443 سرکوب شد، روی UDP/443. ترفندهای QUIC initial
    fragmentation.
13. **حمل روی WebSocket/gRPC/XHTTP** — xray از قبل پشتیبانی می‌کند؛ افزودن انتخاب خودکار transport.

## ۴. موتور خودتطبیق (قلب بن‌بست‌ناپذیری)

این مهم‌ترین بخش است. به‌جای یک تکنیک ثابت:

```
StrategyEngine
├── arsenal: لیست همه‌ی تکنیک‌ها (لایه ۱–۴) با پارامترهای قابل‌تنظیم
├── prober: یک اتصال آزمایشی به CONNECT_IP با هر استراتژی → سنجش موفقیت
│     (آیا ServerHello واقعی برگشت؟ آیا RST خورد؟ زمان؟)
├── ranker: امتیازدهی به استراتژی‌ها بر اساس نرخ موفقیت اخیر (sliding window)
├── selector: انتخاب بهترین استراتژی برای هر اتصال، با کمی randomization
└── fallback chain: اگر استراتژی A شکست خورد، خودکار B، C، … را امتحان کن
```

نتیجه: کاربر دکمه‌ای نمی‌زند؛ ابزار **خودش** کشف می‌کند کدام ترفند علیه DPI فعلی
ISP کار می‌کند و روی آن قفل می‌شود. اگر DPI رفتارش را عوض کرد، prober دوباره کشف می‌کند.

---

## ۵. به‌روزرسانی استراتژی از راه دور (anti-dictation)

- یک فایل `strategies.json` کوچک (نه کد) که آرایه‌ای از استراتژی‌ها و پارامترها را تعریف می‌کند.
- اپ این فایل را از چند منبع mirror (GitHub raw، IPFS، یک endpoint پشت همان bypass) می‌کشد.
- وقتی DPI یک ترفند را در سطح ملی مسدود کرد، فقط `strategies.json` آپدیت می‌شود —
  بدون نیاز به انتشار نسخه‌ی جدید اپ. این یعنی **سرعت واکنش بالا** و سخت‌بودن مسدودسازی.
- امضای دیجیتال فایل برای جلوگیری از tampering.

## ۶. چندپلتفرمی شدن (حذف وابستگی به Windows)

| پلتفرم | روش injection لایه‌ی پایین | وضعیت |
|---|---|---|
| Windows | WinDivert | ✅ موجود |
| Linux | AF_PACKET / NFQUEUE | پورت Go موجود؛ ادغام با StrategyEngine |
| macOS | BPF | پورت Go موجود |
| **Android root** | raw socket + `CAP_NET_RAW` (همان wrong_seq واقعی) | استپ آینده |
| **Android non-root** | فقط لایه ۲/۳ (fragmentation/SNI) — wrong_seq ناممکن | استپ آینده |

نکته‌ی صادقانه (تأیید‌شده در DESIGN.md): wrong_seq روی اندروید non-root ناممکن است،
**اما** لایه‌ی fragmentation روی همان non-root کاملاً کار می‌کند و علیه DPIهای تک‌سگمنتی
(اکثریت DPIهای ارزان) مؤثر است. پس اندروید non-root هم بن‌بست نیست — فقط arsenal محدودتری دارد.

## ۷. تاب‌آوری در برابر سرکوب فعال (active probing & throttling)

DPIهای پیشرفته فقط مسدود نمی‌کنند، بلکه:
- **Active probing:** خودشان به سرور وصل می‌شوند تا ببینند proxy است یا نه → پاسخ ابزار
  باید مثل یک وب‌سرور واقعی باشد (xray REALITY این را حل می‌کند؛ ادغام شود).
- **Throttling:** سرعت را کم می‌کنند نه قطع → تشخیص throttle و چرخش CONNECT_IP/استراتژی.
- **SNI-based RST:** RST تزریق می‌کنند → تشخیص RST جعلی و نادیده‌گرفتن آن (مثل zapret).

## ۹. نتیجه‌گیری — چرا این «بن‌بست‌ناپذیر» است

| تهدید DPI | پاسخ ابزار |
|---|---|
| مسدودکردن wrong_seq | ۱۲+ تکنیک دیگر در arsenal |
| تغییر رفتار DPI | auto-prober خودکار تکنیک جدید را کشف می‌کند |
| امضای ثابت ابزار | randomization در هر اتصال |
| مسدودکردن یک پلتفرم | Windows/Linux/macOS/Android |
| سرکوب در سطح ملی یک ترفند | آپدیت `strategies.json` از راه دور، بدون انتشار اپ |
| active probing | ادغام REALITY |
| RST injection | تشخیص و نادیده‌گرفتن |
| بسته‌شدن کامل TCP/443 | QUIC/UDP + transportهای جایگزین xray |

هیچ تک‌نقطه‌ی شکستی باقی نمی‌ماند. هر مسیر بسته شد، مسیر بعدی خودکار باز می‌شود.
این تعریف عملیاتی «بن‌بست‌ناپذیری» است.
